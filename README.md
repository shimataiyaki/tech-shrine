## 開発・運用フロー

プロダクトに関するご意見・ご要望は、すべて以下のフローで処理します。

```
graph TD
    A[フィードバック発生源] --> B{種類は？}
    B -->|致命的な不具合| C[開発者が即時対応・事後報告]
    B -->|展示内容に関する要望・助言| D[部長が窓口]
    D --> E{部長の判断}
    E -->|採用| F[開発者が実現可能性を検討]
    E -->|不採用| G[部長から提案元へ説明]
    F --> H{技術的・運用的に可能か}
    H -->|可能| I[開発者が対応]
    H -->|不可能| J[開発者から部長へ理由を説明]
```

### フロー運用の原則

1. **窓口の明確化**：開発現場の混乱を避けるため、プロダクトに関するご意見・ご要望は、**部長**を通じて開発者に伝達されます。
2.**簡素化**：部長の判断は「採用する／しない」の二択。技術的詳細は開発者が引き受ける。
3. **緊急時例外**：展示が物理的に停止する致命的な不具合は、開発者の判断で即時修正し、事後報告とする。
4. **フリーズ期間**：文化祭直前の機能追加・仕様変更は、安定運用を優先し、原則として受け付けない（提供条件 第3条第3項に基づく）。
